Сейчас в Python для работы с PostgreSQL очень популярен **SQLAlchemy 2.0**, особенно его **асинхронная версия** в связке с драйвером `asyncpg` . Это считается современным стандартом для создания новых приложений.

Давайте я покажу простой пример, как создать таблицу и поработать с данными, используя этот подход .

### 1. Установка необходимых библиотек

Сначала установите нужные пакеты:

```bash
pip install sqlalchemy asyncpg
```

### 2. Код: Создание таблицы и CRUD-операции

Создайте файл, например `db_example.py`, и напишите следующий код. Это полноценный рабочий пример, который можно запустить.

```python
import asyncio
from sqlalchemy import String, select
from sqlalchemy.ext.asyncio import AsyncAttrs, async_sessionmaker, create_async_engine
from sqlalchemy.orm import DeclarativeBase, Mapped, mapped_column

# --- 1. Настройка подключения ---
# Используем asyncpg для асинхронной работы с PostgreSQL
DATABASE_URL = "postgresql+asyncpg://tracker_user:secure_password@localhost:5432/person_tracker"

# Создаем асинхронный движок
engine = create_async_engine(DATABASE_URL, echo=True) # echo=True покажет SQL-запросы

# Создаем фабрику сессий (главный интерфейс для работы с БД)
async_session = async_sessionmaker(engine, expire_on_commit=False)

# --- 2. Описание модели (таблицы) ---
class Base(AsyncAttrs, DeclarativeBase):
    pass

class User(Base):
    __tablename__ = "users"

    id: Mapped[int] = mapped_column(primary_key=True)
    name: Mapped[str] = mapped_column(String(50))
    email: Mapped[str] = mapped_column(String(100), unique=True)

# --- 3. Функции для работы с данными ---

# Создание таблиц (если их нет)
async def create_tables():
    async with engine.begin() as conn:
        # Создаст все таблицы, которые определены в Base
        await conn.run_sync(Base.metadata.create_all)

# Создание пользователя
async def create_user(name: str, email: str):
    async with async_session() as session:
        async with session.begin():
            new_user = User(name=name, email=email)
            session.add(new_user)
            # session.commit() вызовется автоматически при выходе из session.begin()
        return new_user

# Получение пользователя по ID
async def get_user(user_id: int):
    async with async_session() as session:
        # Выполняем запрос SELECT
        result = await session.execute(select(User).where(User.id == user_id))
        # Получаем первый результат или None
        return result.scalar_one_or_none()

# Обновление email пользователя
async def update_user_email(user_id: int, new_email: str):
    async with async_session() as session:
        async with session.begin():
            user = await session.get(User, user_id)
            if user:
                user.email = new_email
                return True
            return False

# --- 4. Запуск ---
async def main():
    print("Создание таблиц...")
    await create_tables()

    print("\n--- Создание пользователя ---")
    user = await create_user("Alice", "alice@example.com")
    print(f"Создан пользователь: ID={user.id}, Name={user.name}, Email={user.email}")

    print("\n--- Получение пользователя ---")
    fetched_user = await get_user(user.id)
    print(f"Получен пользователь: ID={fetched_user.id}, Name={fetched_user.name}")

    print("\n--- Обновление email ---")
    await update_user_email(user.id, "alice.new@example.com")
    updated_user = await get_user(user.id)
    print(f"Email обновлен: {updated_user.email}")

if __name__ == "__main__":
    asyncio.run(main())
```

### Объяснение кода

1.  **`create_async_engine` и `async_sessionmaker`**: Это основа для настройки асинхронного подключения к вашей базе PostgreSQL .
2.  **Модель `User`**: Класс `User` описывает структуру таблицы `users`. Каждый атрибут с типом `Mapped` — это колонка в таблице.
3.  **`async with session.begin()`**: Этот паттерн гарантирует, что если внутри блока возникнет ошибка, транзакция будет автоматически отменена (`rollback`). При успешном завершении блока изменения будут сохранены (`commit`) .
4.  **Асинхронность**: Все операции с базой данных выполняются с `await`, что позволяет серверу не блокироваться и обрабатывать больше запросов.

### Почему этот стек сейчас популярен?

*   **SQLAlchemy 2.0** — это мощный инструмент, который позволяет работать с базой данных через Python-объекты, а не писать "сырой" SQL, что делает код чище и безопаснее . Он также решает проблемы с безопасностью, например, автоматически защищает от SQL-инъекций .
*   **Async** режим позволяет эффективно использовать ресурсы, особенно в веб-фреймворках вроде FastAPI, которые поддерживают асинхронность "из коробки" .
*   **asyncpg** — это драйвер, который нужен для подключения к PostgreSQL. Он очень быстрый, и в тестах показывал производительность в 5 раз выше, чем старый `psycopg` .

### Важный совет по производительности

Обратите внимание: в примере используется **пул соединений** (он создается автоматически через `create_async_engine`). Создание нового подключения к базе данных — дорогая операция (может занимать 100-200 мс). Пул переиспользует уже открытые соединения, что критически важно для производительности приложения . Не создавайте новое подключение на каждый запрос вручную .

Если вы используете этот код, не забудьте заменить `DATABASE_URL` на свои данные из файла `.env`.
