Great! You already have PostgreSQL running in your Docker Compose setup. Here's how to add Grafana to it:

## 1. Add Grafana Service to Your Docker Compose

Add this section to your `docker-compose.yml` file:

```yaml
  # Grafana
  grafana:
    image: grafana/grafana:latest
    container_name: person_tracker_grafana
    restart: unless-stopped
    environment:
      - GF_SECURITY_ADMIN_USER=${GRAFANA_ADMIN_USER:-admin}
      - GF_SECURITY_ADMIN_PASSWORD=${GRAFANA_ADMIN_PASSWORD:-admin}
      - GF_INSTALL_PLUGINS=grafana-clock-panel,grafana-simple-json-datasource
      - GF_SERVER_ROOT_URL=http://localhost:3000
    ports:
      - "3000:3000"
    volumes:
      - grafana_data:/var/lib/grafana
      - ./grafana/provisioning:/etc/grafana/provisioning:ro
      - ./grafana/dashboards:/etc/grafana/dashboards:ro
    networks:
      - tracker_network
    depends_on:
      postgres:
        condition: service_healthy
```

## 2. Update Your Volumes Section

Add `grafana_data` to your volumes:

```yaml
volumes:
  redis_data:
  yolov8_models:
  postgres_data:  
  pgadmin_data:
  grafana_data:  # Add this line
```

## 3. Create Grafana Configuration Directory Structure

Create these directories and files in your project root:

```
project/
├── grafana/
│   ├── provisioning/
│   │   ├── datasources/
│   │   │   └── datasource.yml
│   │   └── dashboards/
│   │       └── dashboards.yml
│   └── dashboards/
│       └── meter_watch_dashboard.json
```

### Create `grafana/provisioning/datasources/datasource.yml`:

```yaml
apiVersion: 1

datasources:
  - name: PostgreSQL
    type: postgres
    access: proxy
    url: postgres:5432
    database: ${POSTGRES_DB:-person_tracker}
    user: ${POSTGRES_USER:-tracker_user}
    secureJsonData:
      password: ${POSTGRES_PASSWORD:-secure_password}
    jsonData:
      sslmode: disable
      postgresVersion: 1600
      timescaledb: false
```

### Create `grafana/provisioning/dashboards/dashboards.yml`:

```yaml
apiVersion: 1

providers:
  - name: 'Meter Watch Dashboards'
    orgId: 1
    folder: ''
    folderUid: ''
    type: file
    disableDeletion: false
    updateIntervalSeconds: 30
    allowUiUpdates: true
    options:
      path: /etc/grafana/dashboards
      foldersFromFilesStructure: true
```

## 6. Start Grafana

```bash

# Start all services including Grafana
docker-compose up -d

# Or start only Grafana with dependencies
docker-compose up -d postgres grafana
```

   - **Database:** `person_tracker`
   - **User:** `tracker_user`
   - **Password:** `secure_password`
   - **SSL Mode:** `disable`
5. Click **Save & Test**

### Create Dashboard:
1. Click **+** → **Dashboard** → **Add new panel**
2. Write your SQL queries
3. Choose visualization type
4. Save dashboard

## 9. Important: Create Your Table

Make sure your table exists in PostgreSQL. Create an init script:

Create `postgres/init/01_create_table.sql`:

```sql
-- Create the main events table
CREATE TABLE IF NOT EXISTS events (
    id SERIAL PRIMARY KEY,
    timestamp TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    source VARCHAR(50),
    event_type VARCHAR(50),
    payload JSONB,
    value INTEGER,
    flag BOOLEAN
);

-- Create indexes for better performance
CREATE INDEX idx_events_timestamp ON events(timestamp);
CREATE INDEX idx_events_source ON events(source);
CREATE INDEX idx_events_event_type ON events(event_type);

-- Optional: Create a view for meter readings
CREATE OR REPLACE VIEW meter_readings AS
SELECT 
    id,
    timestamp,
    value,
    payload->>'value' as payload_value
FROM events
WHERE source = 'METER' AND event_type = 'READING';

-- Optional: Create a view for person detections
CREATE OR REPLACE VIEW person_detections AS
SELECT 
    id,
    timestamp,
    payload->>'ids' as person_ids,
    flag
FROM events
WHERE source = 'PERSON_DETECTOR' AND event_type = 'PERSON_DETECTED';
```

## 10. Optimized Queries for Your Dashboard

### Meter Reading Time Series:
```sql
SELECT 
    timestamp as time,
    value
FROM events
WHERE 
    source = 'METER' 
    AND event_type = 'READING'
    AND $__timeFilter(timestamp)
ORDER BY timestamp
```

### Person Detection Count:
```sql
SELECT 
    DATE(timestamp) as date,
    COUNT(*) as detections,
    COUNT(DISTINCT payload->>'ids') as unique_persons
FROM events
WHERE 
    source = 'PERSON_DETECTOR' 
    AND event_type = 'PERSON_DETECTED'
    AND $__timeFilter(timestamp)
GROUP BY DATE(timestamp)
ORDER BY date
```

### Combined View:
```sql
SELECT 
    timestamp as time,
    value as meter_reading,
    CASE 
        WHEN source = 'PERSON_DETECTOR' THEN 1 
        ELSE 0 
    END as person_detected
FROM events
WHERE $__timeFilter(timestamp)
ORDER BY timestamp
```

### Person Detection Details:
```sql
SELECT 
    timestamp,
    payload->>'ids' as detected_person_ids,
    flag
FROM events
WHERE 
    source = 'PERSON_DETECTOR' 
    AND event_type = 'PERSON_DETECTED'
    AND $__timeFilter(timestamp)
ORDER BY timestamp DESC
LIMIT 20
```

## 11. Troubleshooting

### If Grafana can't connect to PostgreSQL:
```bash
# Check if PostgreSQL is running
docker-compose ps postgres

# Check logs
docker-compose logs grafana

# Test connection from inside Grafana container
docker-compose exec grafana sh
# Inside container: 
nc -zv postgres 5432
```

### Reset Grafana admin password:
```bash
docker-compose exec grafana grafana-cli admin reset-admin-password admin
```

### Restart Grafana:
```bash
docker-compose restart grafana
```
